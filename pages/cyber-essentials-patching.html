Cyber Essentials patch management: simple process + proof

For SMEs, the requirement isn’t complicated: keep devices and software updated. The hard part is proving it quickly when an assessor/MSP asks — especially under a tender deadline.

security updates patch management evidence

A minimum viable patching policy (1 page)

  • Scope: all company devices (and clear rules for BYOD).
  • Cadence: automatic updates on; critical security updates within X days.
  • Owner: named person (or MSP) responsible for monitoring and exceptions.
  • Exceptions: documented (why, for how long, and mitigation).

Evidence to collect

In-house IT

  • Screenshot: automatic updates enabled (Windows/macOS)
  • Screenshot/export: device management showing update compliance
  • List of key applications and update approach

Using an MSP

  • Ask for a short statement: patching coverage + cadence
  • Ask for an export/report screenshot showing compliance
  • Confirm how they handle devices off-network

Common failure points

  • Updates enabled but users can defer indefinitely
  • Old devices outside management
  • Key apps not included (browsers, PDF tools, etc.)
  • No visibility report when asked
