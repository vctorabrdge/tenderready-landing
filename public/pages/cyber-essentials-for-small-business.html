Cyber Essentials for small business (UK): the tender-first guide

If a customer or tender requires Cyber Essentials, you don’t need a 40-page security programme. You need a repeatable way to (1) implement the basics, (2) assign ownership, and (3) collect evidence fast.

cyber essentials for small business uk tenders

What Cyber Essentials is (in one paragraph)

Cyber Essentials is a UK, government-backed scheme focused on five control areas. The aim is to reduce common cyber attacks by ensuring your organisation has basic security controls in place — and can show evidence that they’re working.

The five areas (and what “good” looks like)

Firewalls & gateways

Securely configured router/firewall, no default passwords, limited exposure to the internet, and clear ownership (MSP vs in-house).

Secure configuration

Devices set up with safe defaults: encryption, screen locks, reduced admin access, and sensible baseline settings.

User access control

Unique accounts, least privilege, quick leavers process, and admin access restricted.

Malware protection

Defender/AV enabled across devices and not easily disabled; basic protection against common threats.

Security updates

Automatic updates + a process for critical patches, with proof that updates are applied in practice.

If you’re 10–50 staff, the common failure mode isn’t “not doing it” — it’s doing it inconsistently and failing to evidence it under deadline pressure.

Evidence: what you’ll actually need to collect

  • MFA enforcement screenshots (especially admin accounts)
  • Device encryption proof (BitLocker/FileVault)
  • AV/Defender enabled screenshots
  • Update/patch settings proof + a short patching policy
  • User list + admin role list + leavers process note
  • Firewall/router ownership and secure configuration proof

Tip: package these as a single handoff pack for your MSP/assessor. It reduces back-and-forth dramatically.

A simple 7-day plan (tender deadline friendly)

  1. Day 1: run the checklist, mark statuses, and identify blockers.
  2. Day 2: lock down MFA + admin accounts.
  3. Day 3: patching and device baselines (encryption, AV).
  4. Day 4: user access control + leavers process.
  5. Day 5: firewall/gateway evidence (often via MSP).
  6. Day 6: collect remaining evidence + write 1-page policies.
  7. Day 7: export your handoff pack and book assessment.
