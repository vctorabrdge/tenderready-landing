Cyber Essentials checklist (UK): practical steps + evidence prompts

If you’re doing Cyber Essentials because a customer or tender requires it, the hardest part is rarely the “rules” — it’s making sure every control is owned, done, and evidenced. This checklist is written for busy UK SMEs (especially construction & trades).

Cyber Essentials checklist Evidence prompts Tender deadlines

What this checklist covers

Cyber Essentials focuses on five areas. For each one below you’ll see:

  • What to implement (plain English)
  • What evidence to collect (screenshots/docs you’ll need)
  • Common failure points (what tends to break under deadline pressure)

1) Firewalls & internet gateways

Implement

  • Ensure your router/firewall is configured securely (no default admin passwords).
  • Only required services are exposed to the internet.
  • Separate guest Wi‑Fi from company systems (where applicable).

Evidence to collect

  • Firewall/router make & model + who manages it (MSP / in‑house).
  • Screenshots/export showing admin access controls and remote management settings.
  • Brief note explaining network segmentation (even if simple).

Construction/trades tip: if your MSP manages the firewall, ask them for a short evidence pack early — don’t leave it to the last week.

2) Secure configuration

Implement

  • Remove/disable unused accounts and software.
  • Make sure devices are configured securely (encryption, screen lock, no shared admin logins).
  • Lock down cloud admin accounts (M365/Google) with MFA.

Evidence to collect

  • Screenshot: device encryption enabled (BitLocker/FileVault).
  • Screenshot: MFA enforced for admin accounts.
  • Short “baseline config” note: how new laptops/phones are set up.

3) User access control

Implement

  • Each person has their own account; avoid shared logins.
  • Leavers are removed quickly; joiners get least privilege.
  • Admin access is restricted to those who genuinely need it.

Evidence to collect

  • Export/list of user accounts (M365/Google) and admin roles.
  • Leavers process note (who disables accounts, how quickly).
  • Password/MFA policy screenshot or written policy (even 1 page).

4) Malware protection

Implement

  • Enable reputable anti-malware on all devices (e.g., Microsoft Defender).
  • Prevent users from disabling protections.
  • Ensure email/web filtering is sensible (where possible).

Evidence to collect

  • Screenshot: Defender/AV active on sample devices.
  • Policy screenshot: tamper protection / settings locked.
  • Short note: how you cover mobiles (MDM or policy).

5) Security update management

Implement

  • Automatic updates for OS and key apps.
  • A process to apply critical updates quickly.
  • Visibility: you can tell what’s patched vs not.

Evidence to collect

  • Patching policy (even a short written process).
  • Screenshots: update settings enabled on devices / MDM.
  • Proof your MSP monitors patch compliance (if outsourced).
